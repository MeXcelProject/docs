FxDialogue.MeXcelTrader.MQL4Wrapper.API.dll

MQL4 API functions. All methods are thread-safe. Any function call may throw APINotAvailable, so caller always must handle it properly. There is no way to handle event when API is about to become unavailable, but yet still available. It just suddenly dies, because of MQL4 imported functions call specifics (see FxDialogue.MeXcelTrader.MQL4Wrapper.MT4Expert for more details).

Events

Start

Raised at the beginning of MQL4 start() function, in its thread context. API can't be called here, because it will end with deadlock. You should call API functions from different thread context.

event EventHandler Start

Event Type

EventHandler

Stop

Raised after MQL4 start() function finished, in context of dedicated thread. API became not available when Stop event is raised, including Stop event processing.

event EventHandler Stop

Event Type

EventHandler
